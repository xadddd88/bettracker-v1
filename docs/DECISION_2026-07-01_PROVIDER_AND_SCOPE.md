## Решение 2 — Скоуп: МУЛЬТИСПОРТ, футбол глубоко первым

Vision расширяется с «футбол + теннис» до **мультиспорта на базе 6 оплаченных API-Sports спортов** (см. таблицу).

**Приоритет беты (Решение 3):**
- **Футбол — ГЛУБОКО:** Scout v2 (calendar-driven) + Analyst v2 (deep) + авто-сеттлмент (1X2). Первый и единственный «глубокий» спорт в бете.
- **Баскетбол / NBA / Ам. футбол / Волейбол / Хоккей — БАЗОВО:** fixtures + результаты + авто-сеттлмент по moneyline/match winner. Без глубокого анализа в бете.
- **Теннис — отдельный трек** (ниже), вне беты api-sports.io.

## Теннис — отдельный трек, ОПЛАЧЕН

**Тенниса нет в api-sports.io** — это отдельный продукт `api-tennis.com` (REST + live WSS). **Статус: ОПЛАЧЕН и активен.** Ведётся отдельным треком (Дима), вне мультиспорт-скоупа api-sports.io (отдельный продукт, отдельный APIkey, своё пространство ID). Интегрируется отдельно; спека live-фида — `docs/TENNIS_TRACK_API_NOTES.md`.

## Последствия для роадмапа (research-уровень, НЕ build-plan)

1. **fixture_id всегда с неймспейсом спорта:** составной ключ `(sport, provider_fixture_id)`. У каждого хоста своё пространство ID.
2. **Scout v2 / Analyst v2 — sport-agnostic ядро + per-sport схемы.** Глубокие входы и рынки различаются по спорту (у футбола форма/xG/H2H; у баскетбола/NBA — иные метрики; у хоккея/AF/волейбола — свои).
3. **Безопасные рынки авто-сеттлмента различаются по спорту** (v1, детерминированные по финальному результату):
   - Футбол → 1X2 (есть ничья)
   - Баскетбол / NBA / Ам. футбол / Хоккей → Moneyline (победитель)
   - Волейбол → Match Winner
   - Форы/тоталы/props → ручная проверка в v1.
4. **Покрытие / задержка результатов / доступность коэффициентов варьируются по спорту и лиге** — опираться на поле `coverage` каждого соревнования; авто-сеттлмент только там, где есть livescore.
5. **Стоимость:** 6 спортов api-sports.io оплачены; теннис (api-tennis.com) оплачен отдельно. Все спорты vision покрыты.
